文档缺失 (5)

#1high

No documented CPU-only build path

来源: docs/en/installing_guide.md
说明
The documentation does not describe how to build the package without the Ascend CANN toolkit. The installing_guide.md only covers container and bare-metal installation on systems with Ascend NPU hardware.
#2medium

No documented CPU-only test path

来源: tests/run_test.sh, tests/run_UT_test.sh
说明
The test scripts (run_test.sh, run_UT_test.sh) have --cpu_only flags but the documentation does not explain what subset of tests can run on CPU or how to set up the environment for CPU-only testing.
#3low

pytest mock not documented

来源: tests/UT/run.py
说明
The UT/run.py file mocks torch_npu at module level, but this mock is not applied when running tests via pytest directly. The documentation does not mention this discrepancy or how to run pytest with the mock.
#4low

requirements.txt not fully documented

来源: requirements.txt vs docs/en/installing_guide.md
说明
The requirements.txt lists many packages (diffusers, transformers, open_clip_torch, etc.) but the installing guide does not explain when each is needed or how they relate to different models.
#5medium

No CONTRIBUTING build/test instructions

来源: contributing.md
说明
contributing.md only contains links to Ascend community guidelines but does not provide project-specific build/test instructions for contributors.

遇到的问题 (4)

Build failed: 'Can not find toolkit path, please set ASCEND_TOOLKIT_HOME'

根因: MindIE-SD is designed for Ascend NPU hardware. The CANN toolkit (Ascend Computing Architecture) is required for building custom operators and plugins. The toolkit is only available on systems with Ascend hardware.

来源: build/build_ops.sh lines 19-31, build/build_plugin.sh lines 15-29

UT run.py failed with 'No module named einops'

来源: mindiesd/layers/flash_attn/sparse_flash_attn_rf_v2.py line 17 imports einops

UT run.py failed with 'No module named safetensors'

来源: mindiesd/quantization/quantize.py line 20 imports safetensors

pytest fails with 'No module named torch_npu' for 17 tests

根因: The mock is defined in UT/run.py which is only used by the unittest runner. pytest collects tests directly without going through run.py. A conftest.py with the same mock would fix this.

来源: tests/UT/run.py (mock definition) vs pytest collection
