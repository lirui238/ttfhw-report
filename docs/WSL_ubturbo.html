文档缺失 (7)

#1README 未说明构建依赖 openEuler 特有文件 /etc/openEuler-release，在 Ubuntu 等非 openEuler 系统上无法直接构建
#2README 未提及 libvirt-dev 是 rmrs 插件的必需构建依赖
#3README 未提及需要构建 libboundscheck 子模块（仅在 openEuler 上预装 /usr/lib64/libboundscheck.so）
#4README 未提及 GCC 版本兼容性要求（代码在 GCC 13 下因缺少 #include <cstdint> 编译失败）
#5README 未提及 python3 是测试构建的必需依赖（mockcpp 需要 Python）
#6README 未提及构建前需要初始化 git 子模块（build.sh 中包含此步骤但未作为前置条件列出）
#7文档中提到 SMAP 需要提前安装，但未说明 SMAP 不可用时二进制能否正常运行

遇到的问题 (8)

Docker 镜像拉取失败 - internal-registry.example.com 无法访问

方案: 使用备用镜像源 docker.1ms.run 拉取 Ubuntu 24.04

来源: Docker 网络配置

CMake 报错 /etc/openEuler-release 不存在

方案: 创建 /etc/openEuler-release 并写入 'openEuler release 22.03'

来源: CMakeLists.txt 第151行 - file(READ "/etc/openEuler-release")

git submodule update --init --recursive 失败 - github.com 无法访问导致 spdlog 克隆失败

方案: 仅初始化构建所需的核心子模块（libboundscheck, rapidjson, googletest, mockcpp）

来源: .gitmodules

securec.h 找不到 - libboundscheck 子模块目录为空

方案: 从 3rdparty/libboundscheck 子模块源码构建 libboundscheck.so 并安装到 /usr/lib64/ 和 /usr/include/

来源: CMakeLists.txt 中 set(BOUNDSCHECK_LIB /usr/lib64/libboundscheck.so)

libvirt/libvirt.h 找不到

方案: 安装 libvirt-dev 包

来源: plugins/rmrs/src/ubturbo_plugin/export/libvirt/rmrs_libvirt_module.h 第16行

GCC 13 下编译失败 - 'uint32_t'/'uint64_t'/'uint16_t' does not name a type

方案: 通过 CMAKE_CXX_FLAGS='-include cstdint' 自动包含 cstdint 头文件，避免修改源码

来源: GCC 13 隐式包含变更 - 代码中多处缺少 #include <cstdint>

测试构建找不到 boundscheck 库

方案: 同时将 libboundscheck.so 安装到 /usr/lib/ 和 /usr/lib/x86_64-linux-gnu/

来源: test/CMakeLists.txt 中 find_library(BOUNDSCHECK_LIB boundscheck)

mockcpp 配置失败 - No Python interpreter found

方案: 安装 python3

来源: test/3rdparty/mockcpp/src/CMakeLists.txt
