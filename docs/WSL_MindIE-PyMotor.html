遇到的问题 (5)

Docker Hub pull timeout

方案: Used locally cached Ubuntu 24.04 image (docker.1ms.run/library/ubuntu:24.04)

来源: Local Docker image cache

pip install from default PyPI index was very slow/timed out from this network location

方案: Configured Aliyun pip mirror (https://mirrors.aliyun.com/pypi/simple/) which resolved packages at 2-7 MB/s

来源: Network conditions in China

Initial pip install hung with zombie pip processes

方案: Killed stuck processes, recreated venv, upgraded pip to latest version, installed packages in batches

来源: Manual troubleshooting

tokenizers package download timeout during transformers installation

方案: pip auto-retried and completed the download on second attempt

来源: pip built-in retry mechanism

Examples/samples cannot be run without NPU hardware and K8s cluster

方案: Noted as not_applicable - this is a deployment-focused project that requires specific hardware

来源: docs/zh/user_guide/environment_preparation.md - requires npu-smi, kubectl, Ascend Docker Runtime, MindCluster components
