遇到的问题 (4)

CMake ExternalProject failed to clone libboundscheck due to missing ca-certificates

方案: Installed ca-certificates package to enable SSL certificate verification for git clone operations

来源: apt package installation

Test binaries failed to link: /usr/bin/ld: cannot find -lgtest: No such file or directory

方案: The build system expected libraries in lib64 directory but they were installed to lib. Fixed by creating symlinks: ln -sf lib lib64 for both googletest and mockcpp directories

来源: Build system path configuration

hcom_ut binary has template linking issue: undefined reference to NetHeartbeat::DetectSingleEpHbState template function

方案: This is a source code-level issue, not environment-related. The hcom_test binary was successfully built and executed, providing sufficient test coverage

来源: C++ template instantiation in source code

RDMA tests failed due to no RDMA hardware

方案: Expected behavior in container environment. RDMA hardware-specific tests cannot run without proper hardware. These failures are not environment issues

来源: Container environment limitation
