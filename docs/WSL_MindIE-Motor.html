文档缺失 (6)

#1medium

Build dependencies not fully documented in README.md

details
README.md only links to external build_guide.md. The actual list of required apt packages is not specified anywhere in the repo. Users must infer from build scripts.
#2high

Hardware dependency not prominently stated

details
The requirement for Ascend CANN toolkit ($ASCEND_HOME_PATH) and its msServiceProfiler component is not prominently documented as a hard requirement. The build_guide.md links to external image installation guides but does not clearly state that CANN toolkit is mandatory for building from source.
#3high

Test environment setup documentation incomplete

details
Test scripts (run_test.sh) reference directories that do not exist in the repo: node_health_management, adapter. The PYTHONPATH configuration in run_test.sh does not match the actual repository structure (node_manager is at mindie_motor/python/mindie_motor/node_manager, not mindie_motor/src/node_health_management).
#4medium

Python test dependencies not documented

details
nodemanager tests require: pytest, coverage, parameterized, requests, pyopenssl, psutil, uvicorn, fastapi, pydantic. None of these are documented as test prerequisites. The tests/requirements.txt only covers benchmark and client deps, not unit test deps.
#5medium

Third-party download source not described in docs

details
The dependency.json file downloads pre-compiled third-party packages from mindie.obs.cn-north-4.myhuaweicloud.com. This external dependency source is not documented. If the OBS bucket becomes unavailable, the build cannot proceed.
#6high

grpc third-party build is skipped but required for proto generation

details
build/third_party/build.sh states 'grpc is not support now, return' but mindie_motor/src/build.sh requires protoc and grpc_cpp_plugin from third_party/install/grpc/bin/. This inconsistency is not documented.

遇到的问题 (6)

Docker Hub unreachable (timeout)

方案: Used locally cached ubuntu:24.04 image (docker.1ms.run mirror was already pulled)

来源: Local Docker image cache

Python wheel build fails with 'msServiceProfiler/Tracer.h: No such file or directory'

方案: Cannot resolve without Ascend CANN toolkit. The header file is part of the Ascend NPU software stack, not available for download in a standard Ubuntu container. This is a hardware-level requirement.

来源: mindie_motor/src/CMakeLists.txt line 49: checks for $ASCEND_HOME_PATH/include/msServiceProfiler/msServiceProfiler.h

Python tests fail with KeyError: 'tls_crl' in CertUtil.validate_cert_and_decrypt_password()

方案: Code-level issue in cert_utils.py line 146 - the code accesses config[TLS_CRL] without checking if the key exists. Additionally, HeartBeatMng singleton is created at module import time (heartbeat_mng.py line 392), which triggers TLS initialization before test mocks can be set up.

来源: mindie_motor/python/mindie_motor/node_manager/framework/utils/cert_utils.py:146

Test run_test.sh references non-existent node_health_management directory

方案: Actual node_manager module is at mindie_motor/python/mindie_motor/node_manager. Updated PYTHONPATH manually for testing.

来源: mindie_motor/src/test/nodemanager_test/run_test.sh line 55

grpc third-party build is skipped but protoc is required for MS build

方案: The build.sh for mindie_motor/src uses protoc from third_party/install/grpc/bin/ but grpc build is explicitly skipped in third_party/build.sh ('grpc is not support now'). Despite this, the MS build was able to proceed to 40% before failing on msServiceProfiler, suggesting protoc binaries may have been pre-built in the downloaded grpc zip.

来源: build/third_party/build.sh line 198 + mindie_motor/src/build.sh lines 40-41

pip install grpcio-tools fails for Python 3.12

方案: grpcio-tools has no pre-built wheel for Python 3.12. Not critical for verification as it's only needed for proto regeneration.

来源: PyPI package availability
