文档缺失 (5)

#1doc/installation.md中未明确列出apt-get需要安装的系统依赖包名称（cmake, gcc, ninja等）
#2文档中推荐的依赖版本（cmake 3.20.x, gcc 11.4+, python 3.11.10）与实际可编译版本存在差异（实际使用gcc 13.3.0, python 3.12, cmake 3.28均编译通过）
#3git子模块克隆失败时没有提供备用镜像源说明（GitHub在国内可能不可达）
#4build_and_pack_run.sh脚本路径在doc/installation.md中直接给出，但子目录 script/run_pkg_maker/make_run.sh 的脚本未提及
#5示例程序执行文档引用路径为 example/examples.md，但实际目录名为 examples/（复数形式）

遇到的问题 (2)

git submodule update --init --recursive 失败，无法连接 github.com:443 - connection timeout

根因: 网络环境无法直接访问GitHub

方案: googletest改用GitCode镜像: git clone --depth 1 --branch v1.12.0 https://gitcode.com/google/googletest.git; mockcpp使用gh-proxy代理: git clone --depth 1 --branch v2.7 https://gh-proxy.com/github.com/sinojelly/mockcpp.git

mockcpp v2.7在GitCode上sinojelly/mockcpp返回403，CSDNopensource/mockcpp也返回403

方案: 使用gh-proxy.com作为GitHub的HTTP代理镜像成功克隆
