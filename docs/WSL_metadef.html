文档缺失 (4)

#1CMakeLists.txt uses -D_FORTIFY_SOURCE=2 in target_compile_definitions which causes -D-D_FORTIFY_SOURCE=2 (duplicate -D prefix)
#2Build documentation does not mention that Release build type has issues with FORTIFY_SOURCE macro definition
#3UT build requires USE_CXX11_ABI to be explicitly set to 0 or 1, but this is not documented in build.md
#4UT build documentation does not mention ASAN_OPTIONS=detect_container_overflow=0 may be needed

遇到的问题 (4)

Git ownership error: 'fatal: detected dubious ownership in repository'

方案: Run git config --global --add safe.directory '*' in container

来源: Git security check

Release build failed with 'macro names must be identifiers' error

方案: Switched to Debug build type which does not enable FORTIFY_SOURCE

来源: cmake/intf_pub_linux.cmake line 18 uses invalid -D_FORTIFY_SOURCE in target_compile_definitions

Device build failed due to cross-compilation toolchain issues

方案: Disabled device build with -DENABLE_BUILD_DEVICE=OFF

来源: CMakeLists.txt - ExternalProject_Add for metadef_device_ep

UT build failed with '_GLIBCXX_USE_CXX11_ABI' undefined

方案: Set USE_CXX11_ABI=0 in cmake configuration: -DUSE_CXX11_ABI=0

来源: cmake/third_party/build/modules/*.cmake - requires explicit USE_CXX11_ABI setting
