文档缺失 (5)

#1libaio-devel 依赖未在文档中列出，仅在 test/llt/CMakeLists.txt 第19行发现（-laio 链接依赖）
#2文档声明硬件要求为 Kunpeng-920 (aarch64)，但 x86_64 平台也能成功编译和运行测试
#3缺少本地单元测试的直接执行说明 — test/run_dt.sh 依赖 hdt（Huawei Developer Toolkit），非标准工具
#4未说明 Maven 作为 Java 部分构建的必要依赖
#5quick_start.md 中操作系统要求写为 'Kunpeng-920 / Kunpeng-920B'（应为 openEuler 版本描述）

遇到的问题 (3)

UT 构建链接失败: /usr/bin/ld: cannot find -laio

方案: 安装 libaio-devel 包

来源: test/llt/CMakeLists.txt 第19行 -laio 依赖

Git 在容器中不可用，build.sh 默认调用 git submodule update

方案: 设置 CI_BUILD=1 环境变量跳过 git submodule 步骤（子模块已在宿主机初始化并通过挂载卷映射）

来源: scripts/build.sh 第200行 CI_BUILD 检查

单元测试 TestFileManager.test_write_read_secondary_key_iterator_return_ok Segmentation fault

方案: 未解决 - 这是测试代码本身的缺陷
