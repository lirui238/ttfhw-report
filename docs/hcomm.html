文档缺失 (4)

#1medium

Missing Dockerfile

说明
Repository does not provide a Dockerfile for build environment setup. Users need to set up environment manually.
#2high

CANN toolkit dependency not clearly documented

说明
Build requires CANN toolkit package (ccec compiler, header files) which is not open source. This dependency should be more prominently documented.
#3high

Missing header files in repository

说明
Required header files (dlog_pub.h, securec.h, ascend_hal_external.h, etc.) are not included in the repository and need to be obtained from CANN toolkit.
#4medium

CMake script robustness issue

说明
hcomm_utils.cmake line 107 has variable comparison issue that can fail under certain conditions.

遇到的问题 (3)

Missing ccec compiler

方案: Install CANN toolkit package which provides this compiler. Note: ccec is a proprietary compiler for Ascend chips.

来源: Build error: /usr/local/Ascend/ascend-toolkit/latest/tools/ccec_compiler/bin/ccec: not found

Missing header files (dlog_pub.h, securec.h, ascend_hal_external.h, ascend_hal.h, hccl_msg.h)

方案: These headers are part of the CANN toolkit package. Install the CANN toolkit to /usr/local/Ascend/ascend-toolkit/latest/

来源: Compilation errors in multiple source files

CMake script variable comparison issue in hcomm_utils.cmake:107

方案: Fix the if statement to use proper variable quoting: if("${PRODUCT_SIDE}" STREQUAL "device") instead of if(${PRODUCT_SIDE} STREQUAL "device")

来源: CMake configuration error when ENABLE_UT=ON
