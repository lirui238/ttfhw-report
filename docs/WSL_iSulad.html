遇到的问题 (8)

仓库 Dockerfile 基于 CentOS 7.6 (arm64 架构)，无法在 x86_64 宿主机上运行

方案: 改用 Ubuntu 22.04 作为备选镜像

来源: 自主决策

Ubuntu apt 源网络连接不稳定导致包下载失败

方案: 配置阿里云镜像源加速下载

来源: 自主决策

缺少关键依赖 libisula (lcr)

方案: 从 gitee.com/openeuler/lcr 克隆源码并构建

来源: docs/build_docs/guide/build_guide_zh.md

缺少关键依赖 clibcni

方案: 从 gitee.com/openeuler/clibcni 克隆源码并构建

来源: docs/build_docs/guide/build_guide_zh.md

缺少关键依赖 libevhtp (Ubuntu 22.04 无此包)

方案: 从 gitee.com/src-openeuler/libevhtp 克隆源码并构建

来源: docs/build_docs/guide/build_guide_zh.md

缺少 grpc_cpp_plugin 程序

方案: 安装 protobuf-compiler-grpc 包

来源: cmake/checker.cmake 检查失败提示

缺少 systemd pkg-config (meson 无法找到 systemd 依赖)

方案: 改用 -Dinit-script=sysvinit 选项绕过 systemd init 脚本

来源: 自主决策

编译失败: isula_fopen 函数未定义

方案: 未解决 - API 版本不兼容问题，libisula 2.1.5 不提供 isula_fopen 函数

来源: 编译错误输出
