文档缺失 (7)

#1Python version requirement: docs specify Python 3.10/3.11, but code compiles with Python 3.12 (used 3.12.3 successfully for most targets)
#2NTFS incompatibility: Build fails on NTFS/WSL cross-filesystem because gRPC source contains a Bazel BUILD file that conflicts with the required 'build' directory. Document should mention ext4 requirement for WSL users.
#3Missing CANN dependency documentation: README.md does not mention that CANN (Ascend NPU SDK) is required for full build. Only found implicitly in CMakeLists.txt conditional logic.
#4No standalone unit test instructions: docs/zh/developer_guide/test.md is a placeholder with no actual test execution steps.
#5Incomplete dependency list in docs: README references external doc site; no inline list of build dependencies. Third-party deps only discoverable via dependency.json.
#6WSL-specific guidance missing: No documentation for building in WSL environment. NTFS issues not documented.
#7health_checker.cpp compilation issue: Produces 0-byte object file in CPU-only build; root cause not documented.

遇到的问题 (5)

Python pip: externally-managed-environment error

方案: Use --break-system-packages flag

来源: Standard Ubuntu 24.04 Python 3.12 behavior (PEP 668)

gRPC third-party build failure: 'build' file (Bazel BUILD) conflicts with 'build' directory on NTFS

方案: Relocate project to ext4 filesystem (/home/<user>/ within WSL). NTFS is case-insensitive and cannot distinguish file 'build' from directory 'build'.

来源: Observed behavior; documented as NTFS limitation

WSL service crash during parallel build (24 jobs on 15Gi RAM)

方案: Reduce parallelism to 8 jobs (MAX_COMPILE_CORE_NUM=8). Successfully completed 17/18 targets.

来源: Resource monitoring; WSL memory exhaustion

mindieservice_daemon linker error: undefined reference to HealthChecker::EnqueueErrorMessage

方案: Unresolved. Root cause: health_checker.cpp.o is 0 bytes. Compilation fails silently, likely due to missing CANN-dependent headers or conditional compilation paths that don't apply in CPU-only mode.

来源: Investigation: nm inspection, manual compilation test, include path analysis

Cannot run examples or tests: require Ascend NPU hardware

方案: N/A - hardware dependency. Full verification requires Ascend NPU device with CANN installed.

来源: CMakeLists.txt ASCEND_HOME_PATH conditional; examples/atb_models structure
