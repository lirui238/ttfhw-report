遇到的问题 (8)

Docker mount caching resolved to wrong repo path

方案: Used unique mount point name '/srcrepo' instead of '/repo' or '/workspace' to avoid Docker volume caching from previous containers.

来源: Docker volume mount behavior

libmcpp wrap file uses SSH URL (git@gitcode.com:openUBMC/libmcpp.git)

方案: Changed to HTTPS URL (https://gitcode.com/openUBMC/libmcpp.git) using sed command.

来源: subprojects/libmcpp.wrap

meson: Dependency 'dbus-1' not found (pkg-config not installed)

方案: Installed pkg-config, libdbus-1-dev, libglib2.0-dev packages.

来源: meson.build:89 - dependency('dbus-1', method: 'pkg-config', required: true)

libmcpp subproject: Dependency 'boost' not found

方案: Installed libboost-all-dev package.

来源: subprojects/libmcpp/meson.build:72

Container OOM killed during bulk package installation (exit code 137)

方案: Installed packages in smaller batches instead of all at once.

来源: N/A

apt-get/dpkg lock held by background auto-setup process

方案: Waited for background process to complete before running own apt-get commands.

来源: N/A

mold linker not installed - fell back to ld.bfd

方案: Build succeeded with ld.bfd. mold is optional optimization per README.

来源: meson.build:49 - message('回退到默认 ld')

6 unit tests failed (4 ABI/service tests with exit 1, 2 SIGSEGV)

方案: Not fixed - these appear to require hardware-specific initialization, dbus, or device tree data not available in container. This is expected for a BMC driver project running in a generic container.

来源: test logs in builddir/meson-logs/testlog.txt
