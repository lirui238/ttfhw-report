Python依赖版本要求

#2

运行时依赖说明

详情
文档未明确说明torchair运行时对Ascend SDK动态库的依赖关系(如libge_runner.so)
#3

UT/ST测试环境要求

详情
文档说明了需要ASCEND_CUSTOM_PATH，但未详细说明需要哪些具体的库文件

遇到的问题 (5)

容器内存不足导致OOM

方案: 使用Ubuntu 24.04替代openEuler

来源: 多次容器崩溃后切换镜像

pip安装torch时触发OOM

方案: 使用--no-cache-dir参数

来源: pip install --no-cache-dir torch

torchair导入缺少packaging/numpy模块

方案: pip install packaging numpy

来源: 运行时错误提示

torchair导入缺少libge_runner.so

方案: 无法解决 - 需要Ascend SDK运行时环境

来源: 预期行为，torchair是NPU专用库

UT C++链接失败

方案: 无法解决 - 需要Ascend SDK的GE库

来源: 链接错误显示缺少ge::Graph等符号

编译成功
UT可执行
ST可执行

inductor UT

TorchAir仓库编译验证通过。UT/ST测试需要Ascend SDK环境：(1) build.sh脚本报错'ASCEND_CUSTOM_PATH必须设置'；(2) C++ UT链接失败，缺少GE库符号；(3) Python ST导入torchair失败，缺少libge_runner.so。inductor_npu_ext子模块的UT测试可在CPU stub模式下运行，2/2测试通过。结论：编译成功，UT/ST测试需要Ascend SDK + NPU硬件。
